Before you file — SCOUT criteria
S
Specific
The feature request describes the exact feature or capability that is missing — not a general pain point.
C
Customer-Centric
The request explains the business impact and the customer's use case, not just the technical ask.
O
Outcome-Based
The request is evidence-based and includes ARR, avoided churn, customer sentiment, or any measurable deal size to quantify impact.
U
Unique
The request avoids duplicates for the same customer. Check if the request already exists for that account — if it does, add context (ARR, urgency, competitor info) to the existing entry rather than creating a new one.
T
Trimmed
The request articulates a single, stand-alone need. If a customer has three different needs, create three separate submissions — this routes each request to the right PM.

Describe the specific business problem the customer faces, the use case behind the ask, and how they intend to use this feature.
Quantify the impact (incremental ARR, revenue at risk, efficiency gains, competitive advantage) and define what success looks like once delivered.
Include any specific technical constraints or requirements (APIs, data formats, integrations, etc.).
When is the customer comfortable adopting this feature?
Describe any workarounds or alternatives currently in use, if available.
